## What sponsorships fund

- **🛠️ Test hardware.** Each new Bluetooth speaker model — IKEA, Sony, JBL, Yandex, no-name brands — has its own BlueZ quirks that can only be debugged on a real device.
- **🌐 Infrastructure** — domain, documentation hosting, CI runners, occasional VPS for the test stand.
- **⏱️ Time** to chase regressions in BlueZ and the broader Linux Bluetooth stack — where most of the unglamorous maintenance work lives.

## FAQ

### What does my donation buy me?

Honestly: nothing exclusive. All features are available to everyone, all source code is public, all releases ship to GitHub the same day. You're sponsoring the project, not buying access.

### Are donations tax-deductible?

No — this is a personal opensource project, not a registered nonprofit. Donations are voluntary contributions, not charitable gifts in any tax-recognized sense.

### My country is not in the supported list of Liberapay / NOWPayments. What now?

The direct crypto addresses above work from anywhere. If crypto isn't an option for you either, drop a note via [GitHub Issues](https://github.com/trudenboy/sendspin-bt-bridge/issues) — we can sort out a SWIFT/Wise transfer for larger one-time contributions.

### Why no Stripe / PayPal / GitHub Sponsors / Buy Me a Coffee?

All of those depend on Stripe for payouts, and Stripe doesn't currently support Kazakhstan or Russia (where the maintainer resides). Working around it would require establishing a US LLC or UK Ltd, which adds compliance overhead disproportionate to opensource donation volumes.

### Can I see how donations are spent?

Spending breakdown is summarized periodically in the [project changelog](https://github.com/trudenboy/sendspin-bt-bridge/blob/main/CHANGELOG.md) and release notes when relevant (e.g., when new test hardware drives a compatibility fix).

## Other ways to help (no money required)

- 🐛 **File good bug reports** — clear repro steps and `docker logs` output save hours of guesswork.
- 🔧 **Send pull requests** — even small fixes (typos, README clarifications, BlueZ quirk workarounds) are appreciated.
- ⭐ **Star the repo** — helps discoverability for others looking for a Bluetooth bridge solution.
- 📣 **Share the project** — Home Assistant forum, Reddit r/homeassistant, Discord servers, your homelab buddies.
- 🌍 **Translate** — the bridge speaks English by default; native-language documentation contributions are welcome.
