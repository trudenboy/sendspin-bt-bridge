`2.40.5` is the first stable release that bundles the full set of March 18 add-on release-engineering improvements into one documented milestone.

## What shipped

### Manual port overrides

Operators can now set top-level `WEB_PORT` and `BASE_LISTEN_PORT` values from configuration surfaces, including the web UI and Home Assistant add-on options.

In Home Assistant add-on mode, the fixed ingress port still stays channel-safe, while `WEB_PORT` opens an additional direct listener only when it is explicitly configured.

### Parallel prerelease tracks on one HAOS host

The release engineering flow now supports clearer `stable`, `rc`, and `beta` separation:

- distinct default ingress ports per track
- distinct default player listen-port ranges per track
- channel-specific branding for prerelease add-ons
- manual startup defaults for prerelease tracks

That makes side-by-side HAOS testing safer without changing the stable add-on defaults.

### Better autogenerated GitHub Releases

GitHub-generated release notes are now treated as optional enrichment instead of the only source of truth.

The release workflow builds the main release body from the matching `CHANGELOG.md` section first, then appends generated notes only when GitHub actually returns useful content.

### Home Assistant ingress and config fixes

The release also includes several HA-facing fixes that were found during live testing:

- album artwork now stays on the active add-on origin when opened through ingress
- solo-player transport commands keep working even when Music Assistant syncgroup discovery is empty
- optional manual add-on ports no longer break Supervisor validation when left unset

## Why this release matters

This milestone was not only about features. It tightened the delivery pipeline itself:

- prerelease channels are easier to test safely
- release notes are now reliable
- HA add-on option handling is stricter and less fragile
- the docs and UI better reflect the actual installed track
