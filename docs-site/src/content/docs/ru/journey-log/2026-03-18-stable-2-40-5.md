`2.40.5` — первый стабильный релиз, в который собран весь пакет улучшений add-on release engineering, сделанных 18 марта.

## Что вошло в релиз

### Ручные port overrides

Теперь можно задавать верхнеуровневые `WEB_PORT` и `BASE_LISTEN_PORT` через конфигурацию, веб-интерфейс и Home Assistant add-on options.

В режиме Home Assistant add-on фиксированный ingress-порт остаётся безопасным для канала установки, а `WEB_PORT` открывает дополнительный прямой listener только если его задали явно.

### Параллельные prerelease-треки на одном HAOS-хосте

Release engineering теперь лучше разводит `stable`, `rc` и `beta`:

- отдельные ingress-порты по трекам
- отдельные базовые диапазоны player listen ports
- channel-specific branding для prerelease-аддонов
- manual startup по умолчанию для prerelease-треков

Это делает side-by-side тестирование на HAOS заметно безопаснее и не меняет дефолты стабильного аддона.

### Более надёжные autogenerated GitHub Releases

Автогенерированные release notes GitHub теперь считаются необязательным дополнением, а не единственным источником правды.

Основное тело релиза сначала собирается из соответствующей секции `CHANGELOG.md`, а GitHub-generated summary добавляется только если GitHub действительно вернул полезный текст.

### Фиксы для Home Assistant ingress и config

В релиз также вошли несколько HA-фиксов, найденных на живой установке:

- обложки альбомов теперь остаются на origin активного аддона при работе через ingress
- transport-команды solo-плееров продолжают работать, даже если discovery syncgroup-ов в Music Assistant пуст
- опциональные ручные add-on порты больше не ломают валидацию Supervisor, если оставить их незаполненными

## Почему этот релиз важен

Это обновление не только про новые возможности. Оно укрепляет сам процесс доставки:

- prerelease-каналы стало безопаснее тестировать
- release notes теперь надёжны
- обработка add-on options в HA стала строже и менее хрупкой
- документация и UI лучше отражают реально установленный трек
