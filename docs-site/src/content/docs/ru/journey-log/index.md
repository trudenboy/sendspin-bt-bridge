- заметки о stable / RC / beta релизах
- изменения в release engineering: GitHub Releases, упаковка add-on и публикация каналов
- архитектурные развилки и сдвиги в развёртывании
- важные багфиксы, найденные на живых установках Home Assistant, Docker и LXC
- короткие объяснения, что изменилось, почему это важно и как это выкатывалось

## Что сюда писать

Используйте Бортжурнал, когда релиз меняет способ поставки проекта, его эксплуатацию или объяснение в документации.

«Вечные» материалы про установку, настройку, API и архитектуру держите в основной документации, а краткие пользовательские релизные пункты — в `CHANGELOG.md`.
