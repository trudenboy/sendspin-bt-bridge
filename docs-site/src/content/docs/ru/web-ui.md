Веб-интерфейс доступен на порту **8080** (Docker/LXC) или через **HA Ingress** (аддон Home Assistant). Данные обновляются в реальном времени через Server-Sent Events — перезагрузка страницы не требуется.

## Обзор

![Полная панель управления с 6 карточками устройств, групповыми элементами управления и шапкой](/sendspin-bt-bridge/screenshots/screenshot-dashboard-full.png)

Панель управления отображает все настроенные Bluetooth-устройства в виде карточек. Карточки сортируются автоматически: сначала воспроизводящие устройства, затем подключённые, но простаивающие, и в конце — неактивные. Устройства из одной группы синхронизации MA остаются рядом.

## Групповые элементы управления

![Панель групповых элементов управления: выпадающий список GROUP, счётчик игроков, флажок All, слайдер VOL 50%, Mute All, Pause All](/sendspin-bt-bridge/screenshots/screenshot-group-controls.png)

Панель управления расположена над карточками. Все операции применяются только к выбранным (отмеченным флажком) устройствам.

| Элемент | Описание |
|---|---|
| **Выпадающий список GROUP** | Фильтр карточек по группе синхронизации MA или «All groups» |
| **Счётчик игроков** | Количество устройств при текущем фильтре |
| **Флажок All** | Отметить / снять все видимые карточки |
| **Слайдер VOL** | Устанавливает громкость на всех выбранных устройствах одновременно |
| **🔈 Mute All** | Переключает mute на всех выбранных устройствах |
| **▮▮ Pause All** | Отправляет паузу один раз на группу синхронизации MA (без рассинхронизации); повторное нажатие — воспроизведение |

<Aside type="tip">
  **Pause All** учитывает группы синхронизации: команда пауза отправляется один раз на каждую группу MA, а не на каждое устройство отдельно. Это сохраняет синхронизацию многоколоночных групп.
</Aside>

## Карточки устройств

Каждая Bluetooth-колонка отображается в отдельной карточке, разделённой на пять колонок.

### Идентификатор устройства (левая колонка)

![Карточка устройства ENEBY20 во время воспроизведения с анимированными полосами эквалайзера](/sendspin-bt-bridge/screenshots/screenshot-device-card-playing.png)

- **Имя плеера** — отображается основным цветом текста, всегда видно
- **Полосы эквалайзера** — четыре анимированных полоски появляются справа от имени при активном воспроизведении и исчезают при остановке
- **Флажок выбора** — включить/исключить устройство из групповых операций
- **Значок группы** — `🔗 GroupName` под именем, если устройство входит в группу синхронизации MA

**При наведении курсора** появляются дополнительные сведения:

![Карточка устройства при наведении: MAC, WebSocket URL, полный MAC адаптера, адрес MA, имя аудиосинка и кнопки управления](/sendspin-bt-bridge/screenshots/screenshot-device-card-hover.png)

- **MAC-адрес** — Bluetooth MAC колонки
- **WebSocket URL** — адрес `ws://`, который использует MA (например, `ws://192.168.10.10:8928/sendspin`)
- **Полный MAC адаптера** — рядом с именем адаптера в колонке подключения
- **Адрес MA сервера** — `host:port` или `auto:9000`
- **Имя аудиосинка** — PulseAudio/PipeWire синк в колонке громкости

### Колонка подключения

Две строки — Bluetooth и Music Assistant:

**Bluetooth:**
| Индикатор | Значение |
|---|---|
| 🟢 Connected | A2DP-соединение установлено |
| 🟡 Reconnecting (N) | Идёт переподключение; N — номер попытки |
| 🔴 Disconnected | Нет Bluetooth-соединения |

Имя адаптера (`hci0`, `hci1`) показано рядом с точкой. При наведении — полный MAC адаптера и MAC устройства.

**Music Assistant:**
| Индикатор | Значение |
|---|---|
| 🟢 Connected | WebSocket-сессия с MA активна |
| 🔴 Disconnected / Error | Нет соединения с MA |

Показывается `host:port` сервера MA (или `auto:9000` при mDNS). При наведении — полный WebSocket URL.

### Колонка воспроизведения

- **Статус** — `▶ Playing` (зелёная точка), `⏸ Stopped` (янтарная точка), `● No Sink` (красный — устройство не подключено)
- **Кнопка паузы/воспроизведения** — `▮▮` при воспроизведении, `▶` при остановке; видна только при активном синке
- **Трек / исполнитель** — отображается под статусом; остаётся на паузе (очищается только когда MA передаёт пустые значения)
  - **Сборники**: при наличии нескольких исполнителей через «/» (например, `Frank Sinatra/Louis Armstrong/Elvis Presley`) отображается `Frank Sinatra +2` — при наведении показывается полный текст
- **Прогресс** — текущая позиция / общая длительность (например, `7:05 / 72:00`)

### Колонка громкости

- **Слайдер громкости** (0–100) — затемнён и заблокирован при отсутствии синка
- **Процент громкости** — рядом со слайдером
- **Кнопка Mute** — 🔈 (включён звук) / 🔇 (отключён, красный фон)
- **Аудиоформат** — мелкий текст с кодеком и параметрами потока (например, `44100Hz/16-bit/2ch`)
- **Имя синка** — при наведении (например, `bluez_sink.FC_58_FA_EB_08_6C.a2dp_sink`); жёлтое предупреждение ⚠, если синк не настроен

### Колонка синхронизации

- **`✓ In sync`** (зелёный) — Music Assistant подтвердил синхронизацию
- **`⚠ Re-anchoring`** (янтарный) — MA активно корректирует тайминг; `Error: N ms` — отклонение при последней коррекции
- **`—`** — устройство остановлено или не подключено

**Счётчик переякорений** — количество коррекций синхронизации с начала потока. Становится **оранжевым** при 10+, **красным** при 100+. Высокое значение при фиксированном значке `delay:` обычно означает, что нужно подобрать `static_delay_ms`.

**Значок задержки** — `delay: -600ms` в колонке синхронизации при воспроизведении, если `static_delay_ms ≠ 0`. Компенсирует буферную задержку A2DP для синхронизации с группой.

### Кнопки управления (при наведении)

| Кнопка | Действие |
|---|---|
| **🔄 Reconnect** | Принудительное переподключение BT без повторного спаривания |
| **🔗 Re-pair** | Полная процедура спаривания (~25 с); переведите устройство в режим спаривания |
| **🔓 Release** | Отключить управление BT для этого устройства; мост перестаёт переподключать |
| **🔒 Reclaim** | Возобновить управление BT (показывается вместо Release после освобождения) |

<Aside type="tip">
  **Release** полезен, когда нужно временно подключить колонку к телефону или ПК. Мост не мешает до нажатия **Reclaim**.
</Aside>

## Сортировка устройств

Карточки автоматически упорядочиваются:

1. **Воспроизводящие** устройства — вверху
2. **Подключённые, но неактивные** — остановленные, но соединённые
3. **Неактивные** — отключённые или освобождённые

Внутри каждого уровня устройства группируются по группе синхронизации MA (по имени группы). Устройства без группы — в конце своего уровня.

## Конфигурация

Раздел **⚙️ Configuration** — сворачиваемая панель в нижней части страницы. Нажмите заголовок, чтобы развернуть.

![Раздел конфигурации с таблицей адаптеров и таблицей устройств с кнопками Add Device и Scan](/sendspin-bt-bridge/screenshots/screenshot-config-adapters.png)

### Основные настройки

- **Bridge name** — необязательный суффикс, добавляемый к каждому имени плеера в MA (вид: `Player @ Name`). Оставьте пустым для отключения.
- **Timezone** — временная зона IANA (например, `Europe/Moscow`). Рядом с выпадающим списком — живые часы.

### Bluetooth-адаптеры

Таблица обнаруженных и добавленных вручную Bluetooth-адаптеров:

| Колонка | Описание |
|---|---|
| **HCI-имя** | Имя интерфейса ядра (`hci0`, `hci1`) |
| **MAC** | Bluetooth MAC адаптера |
| **Name** | Редактируемое удобочитаемое имя |
| **Точка статуса** | Зелёная = адаптер найден и активен |

**↺ Refresh** — повторно обнаружить адаптеры. **+ Add** — добавить адаптер вручную по MAC.

### Bluetooth-устройства

![Таблица устройств с разделами Add Device, Scan и Already Paired](/sendspin-bt-bridge/screenshots/screenshot-config-devices.png)

Одна строка на колонку:

| Колонка | Описание |
|---|---|
| **Player Name** | Имя в Music Assistant |
| **MAC Address** | Bluetooth MAC колонки |
| **Adapter** | Какой BT-адаптер управляет устройством (`default`, `hci0`, `hci1`) |
| **Listen Address** | IP, который плеер MA анонсирует; оставьте пустым для автоопределения |
| **Port** | WebSocket порт этого плеера (по умолчанию — с 8928, увеличивается на 1) |
| **Delay ms** | `static_delay_ms` — отрицательное значение компенсирует буферную задержку A2DP (обычно от `-500` до `-700`) |
| **Format** | Предпочтительный аудиоформат (`flac:44100:16:2` соответствует SBC A2DP без передискретизации) |
| **×** | Удалить устройство |

**+ Add Device** добавляет пустую строку. **🔍 Scan** запускает ~10-секундное фоновое сканирование Bluetooth на всех адаптерах. Нажмите на результат, чтобы заполнить новую строку.

В блоке **Already paired** перечислены ранее спаренные устройства из bluetoothctl. Кнопка **Add** добавляет устройство в таблицу без сканирования.

### Расширенные настройки

Нажмите **▶ Advanced settings** для развёртывания:

- **SENDSPIN_SERVER** — хост/IP сервера MA; `auto` — mDNS-обнаружение
- **SENDSPIN_PORT** — WebSocket порт (по умолчанию 9000)
- **PULSE_LATENCY_MSEC** — подсказка задержки PulseAudio
- **BT_CHECK_INTERVAL** — интервал проверки переподключения в секундах
- **BT_MAX_RECONNECT_FAILS** — максимальное количество подряд идущих неудач при переподключении
- **Prefer SBC codec** — принудительно использовать SBC вместо AAC или aptX

### Сохранение

- **Save Configuration** — записывает `config.json`; вступает в силу после перезапуска
- **Save & Restart** — сохраняет и немедленно перезапускает сервис

<Aside type="caution">
  Изменения конфигурации требуют перезапуска. В режиме аддона Home Assistant аддон перезапускается автоматически при нажатии **Save & Restart**. В Docker/LXC перезапускается процесс Python.
</Aside>

![Раздел диагностики с таблицей состояния системы и зелёными/красными индикаторами](/sendspin-bt-bridge/screenshots/screenshot-diagnostics.png)

Сворачиваемый раздел **Diagnostics** показывает таблицу состояния системы:

| Компонент | Что проверяется |
|---|---|

## Тема оформления

- **Через HA Ingress** — автоматически наследует активную тему Home Assistant (светлую или тёмную) через `postMessage` API
- **Прямой доступ** — следует системной теме браузера (`prefers-color-scheme`)
