Веб-интерфейс доступен на порту **8080** (Docker/LXC) или через **HA Ingress** (аддон Home Assistant). Данные обновляются в реальном времени через Server-Sent Events — перезагрузка страницы не требуется.

## Страница входа

Если `AUTH_ENABLED` включён, мост показывает страницу входа перед доступом к панели управления.

![Страница входа с вкладками Music Assistant и Password](/sendspin-bt-bridge/screenshots/screenshot-login.png)

Доступные методы входа зависят от вашей конфигурации:

| Метод | Когда доступен | Поля |
|---|---|---|
| **Music Assistant** | MA подключён (`MA_API_URL` + `MA_API_TOKEN` настроены) | Имя пользователя + Пароль |
| **Home Assistant** | Работает как аддон HA — аутентификация через login flow HA (поддерживает 2FA) | Имя пользователя + Пароль |
| **Password** | Установлен локальный пароль в конфигурации | Только пароль |

Если доступно несколько методов, вкладки отображаются вверху формы. В режиме аддона HA через Ingress вход автоматический — HA передаёт идентификатор аутентифицированного пользователя.

<Aside type="tip">
  Защита от перебора блокирует страницу входа после **5 неудачных попыток** на **5 минут**.
</Aside>

## Групповые элементы управления

![Панель групповых элементов управления: выпадающий список GROUP, счётчик игроков, флажок All, слайдер VOL 50%, Mute All, Pause All](/sendspin-bt-bridge/screenshots/screenshot-group-controls.png)

Панель управления расположена над карточками. Все операции применяются только к выбранным (отмеченным флажком) устройствам.

| Элемент | Описание |
|---|---|
| **Выпадающий список GROUP** | Фильтр карточек по группе синхронизации MA или «All groups» |
| **Счётчик игроков** | Количество устройств при текущем фильтре |
| **Флажок All** | Отметить / снять все видимые карточки |
| **Слайдер VOL** | Устанавливает громкость на всех выбранных устройствах одновременно |
| **🔈 Mute All** | Переключает mute на всех выбранных устройствах |
| **▮▮ Pause All** | Отправляет паузу один раз на группу синхронизации MA (без рассинхронизации); повторное нажатие — воспроизведение |

<Aside type="tip">
  **Pause All** учитывает группы синхронизации: команда пауза отправляется один раз на каждую группу MA, а не на каждое устройство отдельно. Это сохраняет синхронизацию многоколоночных групп.
</Aside>

### Логика группового слайдера громкости

Групповой **слайдер VOL** использует гибридный подход в зависимости от того, входит ли устройство в группу синхронизации Music Assistant:

**Устройства в группе синхронизации MA** (отмечены значком `🔗 ИмяГруппы`):
- Громкость отправляется через MA API командой `group_volume`, которая применяет **пропорциональный (дельта) алгоритм** MA — уровень каждого участника корректируется относительно текущего значения, сохраняя соотношение между колонками. Например, если Колонка A на 60%, а Колонка B на 40%, сдвиг группового слайдера вниз на 10 установит A на ~50%, а B на ~33%.
- Одна команда `group_volume` отправляется на каждую уникальную группу синхронизации. Если выбранные устройства принадлежат двум группам — обе будут скорректированы.
- Интерфейс обновляется, когда MA возвращает фактические новые значения через протокол sendspin (обычно в течение ~500 мс).

**Устройства вне группы синхронизации** (одиночные плееры, без значка):
- Громкость устанавливается напрямую через PulseAudio (`pactl`) на точное значение слайдера. Одиночное устройство с выставленными 35% будет ровно на 35%.
- Интерфейс обновляется мгновенно.

Это означает, что после изменения группового слайдера устройства в группе синхронизации могут показывать разные проценты (пропорционально), а одиночные устройства — точное значение слайдера.

## Карточки устройств

Каждая Bluetooth-колонка отображается в отдельной карточке, разделённой на пять колонок.

### Идентификатор устройства (левая колонка)

![Карточка устройства ENEBY20 во время воспроизведения с анимированными полосами эквалайзера](/sendspin-bt-bridge/screenshots/screenshot-device-card-playing.png)

- **Имя плеера** — отображается основным цветом текста, всегда видно
- **Полосы эквалайзера** — четыре анимированных полоски появляются справа от имени при активном воспроизведении и исчезают при остановке
- **Флажок выбора** — включить/исключить устройство из групповых операций
- **Значок группы** — `🔗 GroupName` под именем, если устройство входит в группу синхронизации MA

**При наведении курсора** появляются дополнительные сведения:

![Карточка устройства при наведении: MAC, WebSocket URL, полный MAC адаптера, адрес MA, имя аудиосинка и кнопки управления](/sendspin-bt-bridge/screenshots/screenshot-device-card-hover.png)

- **MAC-адрес** — Bluetooth MAC колонки
- **WebSocket URL** — адрес `ws://`, который использует MA (например, `ws://192.168.10.10:8928/sendspin`)
- **Полный MAC адаптера** — рядом с именем адаптера в колонке подключения
- **Адрес MA сервера** — `host:port` или `auto:9000`

### Колонка подключения

Две строки — Bluetooth и Music Assistant:

**Bluetooth:**
| Индикатор | Значение |
|---|---|
| 🟢 Connected | A2DP-соединение установлено |
| 🟡 Reconnecting (N) | Идёт переподключение; N — номер попытки |
| 🔴 Disconnected | Нет Bluetooth-соединения |

Имя адаптера (`hci0`, `hci1`) показано рядом с точкой. При наведении — полный MAC адаптера и MAC устройства.

**Music Assistant:**
| Индикатор | Значение |
|---|---|
| 🟢 Connected | WebSocket-сессия с MA активна |
| 🔴 Disconnected / Error | Нет соединения с MA |

Показывается `host:port` сервера MA (или `auto:9000` при mDNS). При наведении — полный WebSocket URL.

Когда бридж активно получает данные now-playing из MA REST API, рядом с индикатором MA появляется маленький значок **`api`**. Это означает, что для данного устройства доступны полные транспортные кнопки (пред./след./shuffle/repeat), метаданные очереди и обложки альбомов. Значок появляется только при настроенных `MA_API_URL` и `MA_API_TOKEN` и активной доставке данных для конкретного устройства.

### Колонка воспроизведения

- **Статус** — `▶ Playing` (зелёная точка), `⏸ Stopped` (янтарная точка), `● No Sink` (красный — устройство не подключено)
- **Кнопки управления воспроизведением** — видны при активном синке:
  - `◀◀` предыдущий трек, `▮▮` / `▶` пауза/воспроизведение, `▶▶` следующий трек — всегда отображаются при активном синке
  - `⇄` перемешать, `↻` повтор — всегда отображаются при подключённом MA API; отражают актуальное состояние очереди MA и переключают его по нажатию
- **Трек / исполнитель** — отображается под статусом; остаётся на паузе (очищается только когда MA передаёт пустые значения)
  - **Сборники**: при наличии нескольких исполнителей через «/» (например, `Frank Sinatra/Louis Armstrong/Elvis Presley`) отображается `Frank Sinatra +2` — при наведении показывается полный текст
  - **Обложка альбома**: при наведении на строку трека появляется всплывающая миниатюра 120×120 пикселей (из `image_url` в данных MA now-playing). Отображается только при активной MA API-интеграции.
- **Прогресс** — текущая позиция / общая длительность (например, `7:05 / 72:00`)

### Колонка громкости

- **Слайдер громкости** (0–100) — затемнён и заблокирован при отсутствии синка
- **Процент громкости** — рядом со слайдером
- **Кнопка Mute** — 🔈 (включён звук) / 🔇 (отключён, красный фон)
- **Аудиоформат** — мелкий текст с кодеком и параметрами потока (например, `44100Hz/16-bit/2ch`)

Индивидуальные слайдеры громкости используют ту же логику маршрутизации, что и групповой слайдер: при включённом `VOLUME_VIA_MA` и подключённом MA изменения проксируются через MA API. Интерфейс ожидает фактическое значение, возвращённое от MA, перед обновлением. Когда MA недоступен или `VOLUME_VIA_MA` отключён — изменения применяются напрямую через PulseAudio.

### Колонка синхронизации

- **`✓ In sync`** (зелёный) — Music Assistant подтвердил синхронизацию
- **`⚠ Re-anchoring`** (янтарный) — MA активно корректирует тайминг; `Error: N ms` — отклонение при последней коррекции
- **`—`** — устройство остановлено или не подключено

**Счётчик переякорений** — количество коррекций синхронизации с начала потока. Становится **оранжевым** при 10+, **красным** при 100+. Высокое значение при фиксированном значке `delay:` обычно означает, что нужно подобрать `static_delay_ms`.

**Значок задержки** — `delay: -600ms` в колонке синхронизации при воспроизведении, если `static_delay_ms ≠ 0`. Компенсирует буферную задержку A2DP для синхронизации с группой.

**Индикатор заряда** — `🔋 60%` отображается рядом с именем устройства, когда колонка передаёт уровень заряда батареи через интерфейс BlueZ Battery1. Требуется `Experimental = true` в `/etc/bluetooth/main.conf`.

### Кнопки управления (при наведении)

| Кнопка | Действие |
|---|---|
| **ℹ BT Info** | Открывает модальное окно с подробной информацией о Bluetooth-устройстве (MAC, адаптер, имя синка, UUID). Включает кнопку **Copy** для быстрого копирования в баг-репорт |
| **🔄 Reconnect** | Принудительное переподключение BT без повторного спаривания |
| **🔗 Re-pair** | Полная процедура спаривания (~25 с); переведите устройство в режим спаривания |
| **🔓 Release** | Отключить управление BT для этого устройства; мост перестаёт переподключать |
| **🔒 Reclaim** | Возобновить управление BT (показывается вместо Release после освобождения) |

<Aside type="tip">
  **Release** полезен, когда нужно временно подключить колонку к телефону или ПК. Мост не мешает до нажатия **Reclaim**.
</Aside>

## Сортировка устройств

Карточки автоматически упорядочиваются:

1. **Воспроизводящие** устройства — вверху
2. **Подключённые, но неактивные** — остановленные, но соединённые
3. **Неактивные** — отключённые или освобождённые

Внутри каждого уровня устройства группируются по группе синхронизации MA (по имени группы). Устройства без группы — в конце своего уровня.

## Конфигурация

Раздел **⚙️ Configuration** — сворачиваемая панель в нижней части страницы. Нажмите заголовок, чтобы развернуть.

![Раздел конфигурации с таблицей адаптеров и таблицей устройств с кнопками Add Device и Scan](/sendspin-bt-bridge/screenshots/screenshot-config-adapters.png)

### Основные настройки

- **Bridge name** — необязательный суффикс, добавляемый к каждому имени плеера в MA (вид: `Player @ Name`). Оставьте пустым для отключения.
- **Timezone** — временная зона IANA (например, `Europe/Moscow`). Рядом с выпадающим списком — живые часы.

### Bluetooth-адаптеры

Таблица обнаруженных и добавленных вручную Bluetooth-адаптеров:

| Колонка | Описание |
|---|---|
| **HCI-имя** | Имя интерфейса ядра (`hci0`, `hci1`) |
| **MAC** | Bluetooth MAC адаптера |
| **Name** | Редактируемое удобочитаемое имя |
| **Точка статуса** | Зелёная = адаптер найден и активен |

**↺ Refresh** — повторно обнаружить адаптеры. **+ Add** — добавить адаптер вручную по MAC. **↻ Reboot** — перезагрузить конкретный адаптер (`hciconfig down/up`) без перезапуска всего моста.

### Bluetooth-устройства

![Таблица устройств с разделами Add Device, Scan и Already Paired](/sendspin-bt-bridge/screenshots/screenshot-config-devices.png)

Одна строка на колонку:

| Колонка | Описание |
|---|---|
| **▶** | Раскрыть/свернуть строку деталей устройства (Listen Address, Port, Format) |
| **Player Name** | Имя в Music Assistant |
| **MAC Address** | Bluetooth MAC колонки |
| **Adapter** | Какой BT-адаптер управляет устройством (`default`, `hci0`, `hci1`) |
| **Delay (ms)** | `static_delay_ms` — отрицательное значение компенсирует буферную задержку A2DP (обычно от `-500` до `-700`) |
| **×** | Удалить устройство |

**+ Add Device** добавляет пустую строку. **🔍 Scan** запускает ~10-секундное фоновое сканирование Bluetooth на всех адаптерах. Нажмите на результат, чтобы заполнить новую строку. После завершения сканирования кнопка Scan показывает обратный отсчёт до готовности повторного запуска.

В блоке **Already paired** перечислены ранее спаренные устройства из bluetoothctl. Кнопка **Add** добавляет устройство в таблицу без сканирования.

### Настройки Bluetooth

Под таблицей устройств — дополнительные параметры Bluetooth:

| Поле | По умолчанию | Описание |
|---|---|---|
| **BT check interval (s)** | `10` | Как часто проверять соединение каждого устройства |
| **Auto-disable threshold** | `0` | Отключить устройство после N подряд неудачных переподключений; `0` = никогда |
| **Prefer SBC codec** | вкл. | Принудительно SBC после каждого подключения BT — снижает нагрузку на слабом железе (требует PulseAudio 15+) |

### Интеграция с Music Assistant

Раздел **🎵 Music Assistant** в панели конфигурации:

![Интеграция с Music Assistant: статус подключения, настройки сервера и авторизация](/sendspin-bt-bridge/screenshots/screenshot-advanced-settings.png)

| Элемент | Описание |
|---|---|
| **Индикатор статуса** | ⚪ Не подключён / ✅ Подключён — http://ip:port — текущее состояние подключения к MA API. Нажмите **Reconfigure** для изменения. |
| **MA server** | IP/имя хоста, или `auto` для mDNS-обнаружения |
| **MA WebSocket port** | WebSocket-порт Sendspin (по умолчанию `9000`) |
| **PulseAudio latency (ms)** | Большее значение снижает прерывания на медленном железе (по умолчанию `200`). Расположен в разделе General. |
| **🏠 Sign in with Home Assistant** | Кнопка в один клик (только в режиме аддона) — создаёт долгоживущий MA API токен через HA Ingress. Не требует ввода учётных данных. |
| **Advanced: manual JWT token** | Раскрываемый блок `<details>` для Docker/LXC — вставьте URL и токен MA API вручную. Создайте токен в MA → Settings → Profile → Long-lived access tokens. |
| **Route volume through MA** | При включении изменения громкости проходят через MA API — слайдеры в интерфейсе MA остаются синхронизированными |
| **Маршрутизация mute через MA** | Если включено, mute/unmute отправляется через MA API — синхронизирует состояние mute с интерфейсом MA. Если выключено, mute применяется напрямую через PulseAudio |

### Резервное копирование конфигурации

- **⬇ Download** — скачивает текущий `config.json` как вложение с именем файла, содержащим метку времени (например, `config-2026-03-15T10-30-00.json`)
- **⬆ Upload** — загружает ранее скачанный `config.json`. Файл проверяется как валидный JSON перед сохранением. Конфиденциальные ключи (`AUTH_PASSWORD_HASH`, `SECRET_KEY`, `MA_API_TOKEN`) сохраняются из текущей конфигурации и не перезаписываются загруженным файлом.

### Сохранение

- **Save** — записывает `config.json`; вступает в силу после перезапуска
- **Save & Restart** — сохраняет и немедленно перезапускает сервис

После нажатия **Save & Restart** вверху страницы появляется баннер прогресса с фазами перезагрузки:

1. **Saving** — запись `config.json`
2. **Stopping** — остановка текущего сервиса
3. **Waiting** — ожидание перезапуска сервиса
4. **Initializing** — мониторинг готовности подсистем каждого устройства

Фаза инициализации показывает компактную строку статуса:

```
BT 1/2 · PA 0/2 · SS 0/2 · MA … — 5s
```

- **BT** — Bluetooth-соединения установлены
- **PA** — PulseAudio sinks найдены
- **SS** — подпроцессы Sendspin подключены
- **MA** — Music Assistant API подключён

Нажмите на баннер, чтобы развернуть детальную информацию ✅/⏳ по каждой подсистеме каждого устройства.

<Aside type="caution">
  Изменения конфигурации требуют перезапуска. В режиме аддона Home Assistant аддон перезапускается автоматически при нажатии **Save & Restart**. В Docker/LXC перезапускается процесс Python.

## Диагностика

![Раздел диагностики с таблицей состояния системы и зелёными/красными индикаторами](/sendspin-bt-bridge/screenshots/screenshot-diagnostics.png)

Сворачиваемый раздел **Diagnostics** показывает таблицу состояния системы:

| Компонент | Что проверяется |
|---|---|
| Bluetooth daemon | Работа `bluetoothctl` / `bluetoothd` |
| D-Bus socket | Доступность D-Bus сокета |
| Audio server | Работа `pulseaudio` или `pipewire` |
| Adapter hci0 / hci1 | Наличие адаптера, MAC адрес |
| BT audio sinks | Список активных синков `bluez_sink.*` или `bluez_output.*` |
| Строки устройств | Статус подключения + имя синка; красная точка — не подключено |
| **Enabled/Disabled** | Метка «Disabled» оранжевым цветом рядом с отключёнными устройствами, которые были автоматически деактивированы |
| **MA API** | Статус подключения к Music Assistant API и URL |

**↻ Refresh** — повторно запустить проверки.

## Журналы

![Раздел журналов с моноширинным выводом и цветными строками, кнопки фильтра](/sendspin-bt-bridge/screenshots/screenshot-logs.png)

Сворачиваемый раздел **📋 Logs** показывает последние 150 строк журнала приложения.

| Элемент | Описание |
|---|---|
| **Refresh Logs** | Загрузить последние записи журнала |
| **Auto-Refresh: Off/On** | Автоматическое обновление каждые 2 секунды |
| **All / Error / Warning / Info** | Фильтр строк по уровню важности |
| **Log Level** | Изменение уровня логирования (`INFO` / `DEBUG`) без перезагрузки. Нажмите **Apply** для применения ко всем подпроцессам |

Цветовое кодирование строк:
- **Красный** — ERROR
- **Янтарный** — WARNING
- **Белый** — INFO
- **Серый** — DEBUG

## Горячие клавиши

| Клавиша | Действие |
|---|---|
| `R` | Обновить журналы |
| `P` | Pause All |
| `S` | Сохранить конфигурацию |

## Тема оформления

- **Через HA Ingress** — автоматически наследует активную тему Home Assistant (светлую или тёмную) через `postMessage` API
- **Прямой доступ** — следует системной теме браузера (`prefers-color-scheme`)
